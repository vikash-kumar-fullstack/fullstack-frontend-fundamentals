----------------------------------------------------------
1. WHAT IS NPM
----------------------------------------------------------

NPM (Node Package Manager) is the default package manager for Node.js.

Explanation:
It allows developers to install, manage, and share reusable libraries called packages.

Functions of NPM:

Install libraries
Manage project dependencies
Run scripts
Publish packages

Example:

npm install express


----------------------------------------------------------
2. WHAT IS A PACKAGE
----------------------------------------------------------

A package is a reusable piece of code distributed through npm.
It is a collection of modules bundled together and distributed using npm.

Examples:

express → web framework
mongoose → MongoDB ODM
lodash → utility library
dotenv → environment variables

Packages allow developers to avoid writing everything from scratch.


----------------------------------------------------------
3. WHAT IS npm init
----------------------------------------------------------

npm init is used to create a new Node.js project.

Command:

npm init

This command creates a package.json file.


Example interactive steps:

package name
version
description
entry point
author
license


Quick initialization:

npm init -y

This automatically generates package.json with default values.


----------------------------------------------------------
4. WHAT IS package.json
----------------------------------------------------------

package.json is the configuration file of a Node.js project.

Explanation:
It stores metadata about the project and lists all dependencies required to run the application.

Example structure:

{
 "name": "my-app",
 "version": "1.0.0",
 "description": "Node project",
 "main": "index.js",
 "scripts": {
   "start": "node index.js"
 },
 "dependencies": {
   "express": "^4.18.2"
 }
}


----------------------------------------------------------
5. IMPORTANT FIELDS IN package.json
----------------------------------------------------------

name
Name of the project.

version
Current version of the application.

description
Short explanation of the project.

main
Entry file of the application.

Example:
index.js

keyword
used in search engine optimisation

scripts
Commands that can be executed using npm.


dependencies
Libraries required to run the application.


devDependencies
Libraries required only during development.


----------------------------------------------------------
6. DEPENDENCIES
----------------------------------------------------------

Dependencies are packages required for the application to run.

Example:

npm install express

This adds express to dependencies.


package.json:

"dependencies": {
 "express": "^4.18.2"
}


----------------------------------------------------------
7. DEV DEPENDENCIES
----------------------------------------------------------

Dev dependencies are packages used only during development.

Examples:

nodemon
eslint
jest

ESLint: A static code analysis (linting) tool that checks JavaScript code for errors, bad practices, and style issues. It helps maintain consistent and high-quality code during development.

Jest: A JavaScript testing framework used to write and run automated tests to verify that application code works correctly and behaves as expected.

Installation command:

npm install nodemon --save-dev


Example:

"devDependencies": {
 "nodemon": "^3.0.0"
}


----------------------------------------------------------
8. node_modules
----------------------------------------------------------

node_modules is the folder where npm installs packages.

Explanation:

When you install a package:

npm install express

The express package is downloaded into node_modules.


Important point:

node_modules should NOT be pushed to GitHub.


Reason:
It can be recreated using package.json.


----------------------------------------------------------
9. package-lock.json
----------------------------------------------------------

package-lock.json records the exact versions of installed packages.

Purpose:

Ensure consistent installations across environments.


Example:

If two developers run:

npm install

Both will get the exact same dependency versions.


----------------------------------------------------------
10. SCRIPTS IN package.json
----------------------------------------------------------

Scripts allow running commands using npm.

Example:

"scripts": {
 "start": "node index.js",
 "dev": "nodemon index.js"
}


Running scripts:

npm start
npm run dev


----------------------------------------------------------
11. GLOBAL VS LOCAL INSTALLATION
----------------------------------------------------------

Local installation:

npm install express

Installed inside project.


Global installation:

npm install -g nodemon

Installed globally on system.


Check global packages:

npm list -g --depth=0


----------------------------------------------------------
12. SEMANTIC VERSIONING
----------------------------------------------------------

Node packages follow semantic versioning.

Format:

MAJOR.MINOR.PATCH

Example:

4.18.2


Meaning:

MAJOR → breaking changes
MINOR → new features
PATCH → bug fixes


----------------------------------------------------------
13. VERSION SYMBOLS
----------------------------------------------------------

Caret (^)

^4.18.2

Allows updates for minor and patch versions.


Tilde (~)

~4.18.2

Allows only patch updates.


----------------------------------------------------------

----------------------------------------------------------

Explanation:

Reads package.json and installs all dependencies.

----------------------------------------------------------
19. COMMON INTERVIEW QUESTIONS
----------------------------------------------------------

Q1: What is package.json?

Answer:
It is the configuration file that stores metadata and dependencies of a Node.js project.


Q2: Difference between dependencies and devDependencies?

dependencies → required for running application

devDependencies → required only for development.


Q3: Why do we use package-lock.json?

Answer:
To ensure consistent dependency versions across environments.


Q4: Why node_modules is not pushed to GitHub?

Answer:
Because it can be recreated using npm install.
