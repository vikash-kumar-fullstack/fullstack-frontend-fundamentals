------------------------------------------------------------
1. TCP (Transmission Control Protocol)
------------------------------------------------------------

Layer:
Transport Layer

TCP is a connection-oriented protocol used for reliable data transmission.


MAIN FEATURES

Reliable communication

Error checking

Data arrives in correct order

Retransmission of lost packets

Flow control

Congestion control


Because of reliability, protocols like HTTP and SMTP use TCP.



------------------------------------------------------------
TCP CONNECTION FLOW
(VERY IMPORTANT FOR INTERVIEWS)
------------------------------------------------------------

TCP uses something called a

3-WAY HANDSHAKE


STEP 1 — SYN

Client sends synchronization request to server.

Client → Server
SYN


STEP 2 — SYN + ACK

Server acknowledges request.

Server → Client
SYN ACK


STEP 3 — ACK

Client confirms connection.

Client → Server
ACK


Connection established.



------------------------------------------------------------
TCP TERMINATION (4 WAY HANDSHAKE)
------------------------------------------------------------

STEP 1
Client → FIN

STEP 2
Server → ACK

STEP 3
Server → FIN

STEP 4
Client → ACK


Connection closed.



------------------------------------------------------------
WHY TCP IS RELIABLE

Sequence numbers

Acknowledgements

Retransmission

Flow control window


------------------------------------------------------------
INTERVIEW QUESTIONS

Q: Why HTTP uses TCP?

Because HTTP needs reliable delivery of web data.


------------------------------------------------------------
2. UDP (User Datagram Protocol)
------------------------------------------------------------

Layer:
Transport Layer

UDP is a connectionless protocol.


MAIN FEATURES

Fast

Low overhead

No connection setup

No delivery guarantee

No ordering guarantee


UDP is used where speed matters more than reliability.



------------------------------------------------------------
UDP FLOW
------------------------------------------------------------

Client sends packet directly.

No handshake.

Server may receive or lose packets.



------------------------------------------------------------
REAL WORLD USE

Online gaming

Video streaming

DNS queries

VoIP calls



------------------------------------------------------------
TCP vs UDP
------------------------------------------------------------

TCP

Reliable
Connection oriented
Slow


UDP

Fast
Connectionless
Unreliable



------------------------------------------------------------
3. DNS (Domain Name System)
------------------------------------------------------------

Layer:
Application Layer

DNS converts domain names into IP addresses.


Example

google.com → 142.250.183.14


Computers communicate using IP addresses,
but humans remember domain names.



------------------------------------------------------------
DNS QUERY FLOW
------------------------------------------------------------

User enters:

google.com


STEP 1

Browser checks local cache.



STEP 2

OS checks DNS cache.



STEP 3

Request sent to ISP DNS server.



STEP 4

Recursive DNS queries start.



ROOT SERVER

Finds TLD server (.com)



TLD SERVER

Finds authoritative server.



AUTHORITATIVE SERVER

Returns IP address.



STEP 5

Browser receives IP and connects to server.



------------------------------------------------------------
DNS PORT

53


Usually uses UDP

Sometimes TCP for large responses.



------------------------------------------------------------
4. HTTP (HyperText Transfer Protocol)
------------------------------------------------------------

Layer:
Application Layer


HTTP is the protocol used for transferring web pages
between client and server.



DEFAULT PORT

80



------------------------------------------------------------
HTTP REQUEST STRUCTURE
------------------------------------------------------------

GET /products HTTP/1.1

Host: example.com
User-Agent: Chrome


------------------------------------------------------------
HTTP RESPONSE STRUCTURE
------------------------------------------------------------

HTTP/1.1 200 OK

Content-Type: text/html


<html>Page</html>



------------------------------------------------------------
HTTP METHODS

GET

Fetch data


POST

Send data


PUT

Update resource


DELETE

Remove resource



------------------------------------------------------------
HTTP FLOW

STEP 1

Browser sends HTTP request


STEP 2

Server processes request


STEP 3

Server returns response


------------------------------------------------------------
HTTP LIMITATION

No encryption

Data visible in network



------------------------------------------------------------
5. HTTPS (Secure HTTP)
------------------------------------------------------------

HTTPS is HTTP with encryption.

Encryption uses SSL/TLS.



PORT

443



------------------------------------------------------------
HTTPS FLOW
------------------------------------------------------------

STEP 1

TCP handshake


STEP 2

TLS handshake


STEP 3

Certificate verification


STEP 4

Encrypted communication begins.



TLS handshake includes

Key exchange

Cipher negotiation

Authentication

------------------------------------------------------------

Used for transferring files between client and server.

------------------------------------------------------------
FTP WORKING

STEP 2

STEP 3

Commands sent.



STEP 4

Files transferred.



Commands examples

LIST
UPLOAD
DOWNLOAD
