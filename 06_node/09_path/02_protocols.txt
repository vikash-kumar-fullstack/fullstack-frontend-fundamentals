------------------------------------------------------------
1. TCP (Transmission Control Protocol)
------------------------------------------------------------

Layer:
Transport Layer

TCP is a connection-oriented protocol used for reliable data transmission.


MAIN FEATURES

Reliable communication

Error checking

Data arrives in correct order

Retransmission of lost packets

Flow control

Congestion control


Because of reliability, protocols like HTTP and SMTP use TCP.



------------------------------------------------------------
TCP CONNECTION FLOW
(VERY IMPORTANT FOR INTERVIEWS)
------------------------------------------------------------

TCP uses something called a

3-WAY HANDSHAKE


STEP 1 — SYN

Client sends synchronization request to server.

Client → Server
SYN


STEP 2 — SYN + ACK

Server acknowledges request.

Server → Client
SYN ACK


STEP 3 — ACK

Client confirms connection.

Client → Server
ACK


Connection established.



------------------------------------------------------------
TCP TERMINATION (4 WAY HANDSHAKE)
------------------------------------------------------------

STEP 1
Client → FIN

STEP 2
Server → ACK

STEP 3
Server → FIN

STEP 4
Client → ACK


Connection closed.



------------------------------------------------------------
WHY TCP IS RELIABLE

Sequence numbers

Acknowledgements

Retransmission

Flow control window


------------------------------------------------------------
INTERVIEW QUESTIONS

Q: Why HTTP uses TCP?

Because HTTP needs reliable delivery of web data.


------------------------------------------------------------
2. UDP (User Datagram Protocol)
------------------------------------------------------------

Layer:
Transport Layer

UDP is a connectionless protocol.


MAIN FEATURES

Fast

Low overhead

No connection setup

No delivery guarantee

No ordering guarantee


UDP is used where speed matters more than reliability.



------------------------------------------------------------
UDP FLOW
------------------------------------------------------------

Client sends packet directly.

No handshake.

Server may receive or lose packets.



------------------------------------------------------------
REAL WORLD USE

Online gaming

Video streaming

DNS queries

VoIP calls
