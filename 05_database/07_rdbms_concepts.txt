1. WHAT IS RDBMS?
RDBMS is a database system that stores data in the form of tables with relationships.

Explanation:
RDBMS organizes data into tables (relations) where each table consists of rows and columns. It allows linking tables using keys, making data structured and easy to manage.

Examples:
- MySQL
- PostgreSQL
- Oracle

Real-life Example:
Users table connected with Orders table in an e-commerce app.


------------------------------------------------------------


2. WHAT IS A TABLE?
A table is a collection of related data organized in rows and columns.

Explanation:
Each table represents a specific entity like users, products, or orders. Tables are the basic building blocks of RDBMS.

Example:

Users Table:
ID   Name   Age
1    Vikash 22


------------------------------------------------------------


3. WHAT IS A ROW (TUPLE)?
A row represents a single record in a table.

Explanation:
Each row contains complete information about one entity.

Example:
(1, Vikash, 22) → One user


------------------------------------------------------------


4. WHAT IS A COLUMN (ATTRIBUTE)?
A column represents a property or field of data.

Explanation:
Each column stores a specific type of data.

Example:
Name, Age, ID are columns in a table.


------------------------------------------------------------


5. WHAT IS A PRIMARY KEY?
A primary key uniquely identifies each row in a table.

Explanation:
- Must be unique
- Cannot be NULL

Example:
User ID in Users table

Why important:
Ensures no duplicate records.


------------------------------------------------------------


6. WHAT IS A FOREIGN KEY?
A foreign key is a column that creates a link between two tables.

Explanation:
It refers to the primary key of another table.

Example:

Users Table:
ID = 1

Orders Table:
user_id = 1 → connects to Users table

Why important:
Maintains relationships between tables.


------------------------------------------------------------


7. WHAT IS A RELATIONSHIP?
A relationship defines how two tables are connected.

Types:

1. One-to-One
   One user → One profile

2. One-to-Many
   One user → Many orders

3. Many-to-Many
   Many students ↔ Many courses

Explanation:
Relationships are implemented using foreign keys.
