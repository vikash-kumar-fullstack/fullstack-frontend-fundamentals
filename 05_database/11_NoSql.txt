1. WHAT IS NOSQL?
NoSQL is a class of databases designed to handle large-scale, distributed, and flexible data storage.

Explanation:
Unlike traditional relational databases, NoSQL databases do not rely on fixed table structures and rigid schemas. They are designed to handle massive volumes of data, high user traffic, and distributed environments across many machines.

Key Idea:
NoSQL focuses on scalability, flexibility, and performance rather than strict relational structure.

Examples:
MongoDB
Cassandra
Redis
DynamoDB

Real-world Example:
Social media platforms store huge volumes of posts, likes, and comments. This type of rapidly growing and flexible data fits better in NoSQL databases.


------------------------------------------------------------


2. WHY NOSQL WAS INTRODUCED
NoSQL was introduced to solve the limitations of traditional relational databases at large scale.

Problems with RDBMS at scale:
- Difficult horizontal scaling
- Rigid schema
- Performance issues with large joins
- Handling massive distributed systems

Example:
Facebook or Instagram cannot store billions of user interactions efficiently in a single relational server.


------------------------------------------------------------


3. WHAT IS A DISTRIBUTED SYSTEM?
A distributed system is a group of multiple computers working together as a single system.

Explanation:
Instead of storing data on one machine, distributed systems store data across many machines connected through a network.

Benefits:
- Higher scalability
- Better reliability
- Fault tolerance

Example:
Instead of one server storing all user data, multiple servers store different portions of the data.


------------------------------------------------------------


4. WHAT IS A CLUSTER?
A cluster is a group of servers (nodes) that work together to manage data and handle requests.

Explanation:
Each server in the cluster contributes computing power and storage. If one node fails, other nodes can continue working.

Example:
A database cluster may contain 10 servers storing different parts of the data.


------------------------------------------------------------


5. WHAT IS A NODE?
A node is an individual server within a distributed system or cluster.

Explanation:
Each node stores part of the data and processes requests from users.

Example:
In a cluster of 5 machines, each machine is called a node.


------------------------------------------------------------


6. WHAT IS DATA PARTITIONING?
Data partitioning is the process of splitting large data into smaller parts and distributing them across multiple nodes.

Explanation:
Instead of storing all data on one server, data is divided into partitions so different servers handle different data.

Example:
Server 1 → Users A–F
Server 2 → Users G–M
Server 3 → Users N–Z

Benefit:
Improves scalability and performance.


------------------------------------------------------------


7. NETWORK PARTITIONING (PARTITION FAILURE)
Network partition occurs when communication between nodes in a distributed system is broken.

Explanation:
Sometimes network issues prevent nodes from communicating with each other. When this happens, the system becomes divided into separate groups that cannot coordinate.

Example:
Cluster with 5 nodes:
2 nodes cannot communicate with the remaining 3 nodes.

This creates a network partition.


------------------------------------------------------------


8. WHAT IS REPLICATION?
Replication is the process of copying data across multiple servers.

Explanation:
Replication ensures that multiple nodes have the same copy of the data. This improves reliability and availability.

Example:
User data stored on Node A is also copied to Node B and Node C.


------------------------------------------------------------


9. TYPES OF REPLICATION

Primary-Replica Replication:
One server handles writes, and other servers copy the data.

Example:
Primary Node → Accepts writes
Replica Nodes → Copy data

Multi-Master Replication:
Multiple servers can accept writes and replicate data to each other.

Example:
Two servers in different regions both accept updates.


------------------------------------------------------------


10. WHY REPLICATION IS IMPORTANT
Replication improves system availability and fault tolerance.

Benefits:
- Data backup
- High availability
- Disaster recovery
- Load distribution


------------------------------------------------------------


11. REPLICATION PROBLEMS

Data Inconsistency:
Different replicas may temporarily have different data.

Replication Lag:
Replica nodes receive updates later than the primary node.

Conflict Resolution:
When multiple nodes update the same data simultaneously.

Example:
Two users updating the same profile on different servers.
