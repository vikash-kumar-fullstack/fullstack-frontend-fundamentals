
1. WHAT IS JSON?
JSON (JavaScript Object Notation) is a lightweight data format used to store and exchange data.

Explanation:
JSON is widely used in web applications to send and receive data between client and server. It is language-independent and easy for humans and machines to read.

Structure Example:

{
  "name": "Vikash",
  "age": 22,
  "skills": ["JavaScript", "MongoDB"]
}

Key Characteristics:
- Data stored as key-value pairs
- Keys must be strings
- Supports arrays and nested objects

Use Cases:
- API responses
- Configuration files
- Data storage in NoSQL databases


----------------------------------------------------------

2. WHAT IS BSON?
BSON (Binary JSON) is a binary representation of JSON used internally by MongoDB.

Explanation:
MongoDB stores data in BSON format instead of plain JSON because BSON supports additional data types and is optimized for faster storage and retrieval.

Extra Data Types in BSON:
- Date
- Binary
- ObjectId
- Decimal128
- Long

Example BSON Structure:

{
  "_id": ObjectId("6512ab3c4d"),
  "name": "Vikash",
  "age": 22
}


----------------------------------------------------------

3. WHAT IS MONGODB?
MongoDB is a NoSQL document-oriented database that stores data in BSON documents.

Explanation:
Instead of tables and rows like relational databases, MongoDB stores data in collections and documents.

Structure Comparison:

SQL                 MongoDB
----------------------------------
Database            Database
Table               Collection
Row                 Document
Column              Field

14. PROJECTION
----------------------------------------------------------
a.  db.users.find({}, {name:1})

Explanation:
Returns only name field.

b.  db.users.find({}, {name:true,address:true})

Explanation:
Returns only name and address of each documents

c. db.users.find({}, {name:false})

Explanation:
Returns all fields except name of each Documents

15. COMPARISON OPERATORS
----------------------------------------------------------
$gt  greater than
$lt  less than
$gte greater or equal
$lte less or equal
$eq  equal
$ne  not equal
$in  for checking in array
$nin  for checking not in array

Example:

db.users.find({age:{$gt:21}})


----------------------------------------------------------
16. LOGICAL OPERATORS
----------------------------------------------------------
$and
$or
$not

Example:
