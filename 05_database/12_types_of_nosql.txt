1. WHAT IS NOSQL DATABASE CLASSIFICATION?
NoSQL databases are classified based on how they store and organize data.

Explanation:
Unlike relational databases that store data in tables, NoSQL databases use different storage models optimized for scalability, flexibility, and distributed systems.

Main Types of NoSQL Databases:
1. Key-Value Databases
2. Document Databases
3. Column-Family Databases
4. Graph Databases


------------------------------------------------------------


2. KEY-VALUE DATABASE
---------------------
A key-value database stores data as pairs of keys and values.

Explanation:
Each key is unique and used to retrieve the corresponding value. The database does not understand the internal structure of the value; it simply stores and retrieves it.

Structure:
Key → Value

Example:
user_101 → {name: "Vikash", age: 22}

The key acts like an identifier, and the value can be anything such as text, JSON, or binary data.


Examples of Key-Value Databases:
Redis
Amazon DynamoDB
Riak


Use Cases:
- Session storage
- Caching systems
- User authentication tokens
- Shopping cart data


Advantages:
- Extremely fast data retrieval
- Simple architecture
- Easy horizontal scaling


Challenges:
- Limited query capability
- Cannot perform complex queries
- Difficult to represent relationships between data


------------------------------------------------------------


3. DOCUMENT DATABASE
A document database stores data in the form of documents.

Explanation:
Documents usually use JSON-like structures where data is stored as key-value pairs inside a document. Each document can have a different structure.

Structure Example:

{
  "name": "Vikash",
  "age": 22,
  "skills": ["JavaScript", "Python"]
}


Examples of Document Databases:
MongoDB
CouchDB
Firestore


Use Cases:
- Web applications
- Content management systems
- E-commerce product catalogs
- User profile systems


Advantages:
- Flexible schema
- Easy to store complex objects
- Natural mapping with programming objects


Challenges:
- Joins are limited compared to relational databases
- Data duplication may occur
- Maintaining consistency can be harder


------------------------------------------------------------


4. COLUMN-FAMILY DATABASE
Column-family databases store data in column groups rather than rows.

Explanation:
Instead of storing all data in rows like relational databases, these databases organize data by columns, allowing efficient storage and retrieval for large datasets.

Structure Example:

UserID | Name | Email | Age

Data is stored in column families rather than entire rows.


Examples of Column-Family Databases:
Apache Cassandra
HBase
Google Bigtable


Use Cases:
- Big data systems
- Analytics platforms
- Logging systems
- IoT data storage


Advantages:
- High scalability
- Efficient for large datasets
- Optimized for distributed systems


Challenges:
- Complex data modeling
- Limited query flexibility
- Harder to design compared to relational databases
