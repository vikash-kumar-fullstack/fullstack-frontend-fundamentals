1. WHAT IS CLASSIFICATION BASED ON NUMBER OF USERS?
---------------------------------------------------
This classification defines databases based on how many users can access them at a time.

Explanation:
Databases can be designed for a single user or multiple users depending on the application requirements like personal use or enterprise systems.


2. TYPES OF DATABASES BASED ON USERS
------------------------------------

1. Single-User Database
2. Multi-User Database


------------------------------------------------------------


3. SINGLE-USER DATABASE
------------------------
A single-user database allows only one user to access the database at a time.

Explanation:
Only one person can read, write, or modify data at any given moment. These databases are usually used in personal or small-scale applications.

Example:
- Personal computer database
- MS Access used by one person

Real-life Example:
A student maintaining personal records on their laptop.

Advantages:
- Simple to design and manage
- No concurrency issues
- Low cost

Disadvantages:
- Not scalable
- Cannot support teamwork
- Limited performance


------------------------------------------------------------


4. MULTI-USER DATABASE
-----------------------
A multi-user database allows multiple users to access and work on the database simultaneously.

Explanation:
Many users can perform operations like reading and writing data at the same time. DBMS manages concurrency to avoid conflicts.

Example:
- Banking system
- E-commerce websites
- Social media platforms

Real-life Example:
Multiple users placing orders on Amazon at the same time.

Types of Multi-User Systems:
1. Concurrent Users
2. Distributed Users (across locations)



------------------------------------------------------------


5. CONCURRENCY CONTROL (IMPORTANT CONCEPT)
------------------------------------------
Concurrency control ensures that multiple users can access the database without conflicts.

Explanation:
When many users try to update the same data, DBMS ensures consistency using techniques like locking and transactions.

Example:
Two users trying to book the same seat.
DBMS ensures only one booking is successful.



7. ADVANTAGES OF MULTI-USER DATABASE
------------------------------------

- Supports multiple users simultaneously
- Efficient data sharing
- Suitable for large applications
- Better resource utilization


------------------------------------------------------------


8. DISADVANTAGES OF MULTI-USER DATABASE
---------------------------------------

- Complex design
- Requires concurrency control
- Higher cost
- Needs strong security mechanisms
